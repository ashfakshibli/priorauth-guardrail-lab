# Demo Data Safety Statement

**PriorAuth Guardrail Lab contains exclusively synthetic, fictional data.**

## What this project IS

- A software engineering portfolio demo
- A synthetic prior authorization workflow simulation
- An AI systems / guardrails showcase using deterministic rules and optional LLM enhancement

## What this project is NOT

- A HIPAA-covered entity or business associate
- A real insurance or health plan system
- Connected to any real EHR, payer, or claims system
- Processing any real patient, member, or provider data

## Synthetic data characteristics

| Field | What you'll find |
|---|---|
| Member IDs | `MBR-1042`, `MBR-2089` — fictional identifiers |
| Provider IDs | `PRV-217`, `PRV-304` — fictional identifiers |
| Age | Age bands (`45-54`, `65+`) — never date of birth |
| Clinical notes | Generic synthetic summaries — never real records |
| Diagnosis codes | Real ICD-10 code formats used in illustrative examples only |
| Service codes | Real CPT/HCPCS formats used in illustrative examples only |
| Payer names | `SYNTH-PLAN-A`, `SYNTH-PLAN-B` — fictional |

## No personal information

This repository does not and will not contain:

- Real patient names, dates of birth, addresses, or phone numbers
- Social Security numbers or other government identifiers
- Real insurance member or provider identifiers
- Real medical records or claims data
- Real email addresses (except the repo owner's public contact)

## If you fork or extend this project

Maintain the synthetic-data boundary. Do not seed real healthcare data even in development branches. If you need to test with data resembling real records, use a separate private repository with appropriate access controls and compliance posture.
